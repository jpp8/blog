IOC

2018-05-07

1、核心概念

1.1、控制反转

  • 将对象的控制和管理权转交给外部（ioc容器），由主动new对象转为外部提供
	graph LR
        a(new A) -->|转为| b
        subgraph IOC
            b(A)
        end

1.2、ioc容器

  • 负责实例化，配置，组装对象，在ioc中这种对象统称bean

配置bean的方式

  • xml，java，注解

1.3、DI

  • 依赖注入：bean和bean之间的依赖关系
    	graph LR
  subgraph IOC
      A -->|依赖| b
  end

配置bean，通过ioc容器获取bean

2、ioc

2.1、配置bean

<bean id="" class=""/>

  • id：bean的唯一标识
  • class：类的全类名
属性 含义
id bean的唯一id
name bean的别名
class bean类型，全类名
scope 单例还是非单例
init-method 生命周期初始化方法
destroy-method 生命周期初销毁方法
autowire 自动装配
factory-method 工厂方法，静态或是实例
factory-bean 实例工厂bean
lazy-init 是否为懒加载

2.2、实例化ioc容器及获取bean对象

①容器的创建

  • ClassPathXmlApplicationContext("applicationContext.xml")
  • FileSystemXmlApplicationContext("applicationContext.xml")

②获取bean对象

  • getBean("bean名称")
  • getBean("bean名称",bean.class)
  • getBean(bean.class)

③BeanFactory创建ioc容器

  • BeanFactory是延迟加载，只有在获取bean对象的时候才会去创建
  • ApplicationContext是立即加载，容器加载的时候就会创建bean对象

2.3、bean的socpe属性

  • singleton默认为单例
  • prototype为非单例
  • bean为单例的意思是在Spring的IOC容器中只会有该类的一个对象
  • bean对象只有一个就避免了对象的频繁创建与销毁，达到了bean对象的复用，性能高

2.4、bean的实例化

graph LR
s(实例化bean)-->k(1.空参构造方法,2.静态工厂,3.实例工厂,4.factoryBean)

对象交给ioc容器去创建，ioc通过四种方式进行创建对象

  • 空参构造方法
  • 静态工厂
  • 实例工厂
  • 实现FactoryBean接口

①构造方法

通过反射调用空参构造来实例化对象

②通过静态工厂

使用命名属性factory-method指定静态工厂方法本身的名称

③实例工厂

通过实例工厂创建对象

④实现FactoryBean接口

2.5、bean的生命周期

  • 初始化容器
    • 创建对象
    • 执行属性注入（set）
    • BeanPostProcessor，后置处理器，前置功能
    • 执行bean初始化方法
    • BeanPostProcessor，后置处理器，后置功能
  • 使用bean
  • 销毁bean

①创建bean–>②set属性–>③后置处理器前置方法–>④init初始化方法–>⑤后置方法–>⑥bean的使用 –>⑦bean的销毁

3、DI

依赖注入是一个过程，描述对象和对象的内在关系，代码更干净，解耦

3.1、构造器注入

  • constructor-arg ref ：引用对应的bean

3.2、set注入

①注入引用数据类型

  • property
  • ref

②注入基本数据类型

  • property
  • value

3.2、自动装配

ioc容器根据bean所依赖的资源去容器中自动查找并装配

自动装配的方式 autowire

  • 按类型：byType
  • 按名称：byName
  • 按构造方法：constructor
  • 不装配：no

3.4、管理第三方bean

加载外部配置文件

  1. resources下创建一个jdbc.properties文件,并添加对应的属性键值对

  2. 在applicationContext.xml中开context命名空间

  3. 在配置文件中使用context命名空间下的标签来加载properties配置文件

  4. 通过${xxxx} 注入属性
